# Exocortex: Руководство по запуску

## Что такое Exocortex?

Exocortex — это персональная система управления знаниями на основе графа знаний и LLM. Система помогает:
- Сохранять и структурировать информацию в виде связанных концептов
- Автоматически извлекать сущности и связи из текста
- Находить противоречия между идеями
- Обнаруживать неочевидные связи между концептами
- Получать напоминания о «забываемых» знаниях

## Требования

- Python 3.11 или выше
- pip (менеджер пакетов Python)
- API ключ для LLM (OpenAI или совместимый)

## Установка

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd exocortex
```

### 2. Создание виртуального окружения

```bash
python -m venv venv
source venv/bin/activate  # Linux/macOS
# или
venv\Scripts\activate  # Windows
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка конфигурации

Скопируйте файл конфигурации и настройте его:

```bash
cp config/.env.example config/.env
```

Отредактируйте `config/.env` и укажите:
- `LLM_API_KEY`: Ваш API ключ для LLM
- `LLM_MODEL`: Модель для использования (например, `gpt-4`)
- `STORAGE_PATH`: Путь для хранения графа знаний (по умолчанию `data/graph`)

## Запуск

### Быстрый старт

```bash
python -m app.main
```

**Примечание**: Для работы LLM-пайплайна требуется API ключ. Без него система использует fallback-режим с эвристическим извлечением сущностей.

### Запуск через Docker (если доступен)

```bash
docker-compose up -d
```

## Ожидаемое поведение

### После запуска система:

1. **Инициализирует граф знаний**
   - Создаёт пустой граф или загружает существующий из файла `.gexf`
   - Инициализирует хранилище фрагментов знаний

2. **Готова к приёму данных**
   - Можно добавлять тексты через API или CLI
   - Система автоматически извлекает сущности и связи
   - Узлы и связи сохраняются в графе

3. **Базовые возможности MVP**:
   - ✅ Добавление узлов и связей вручную
   - ✅ Поиск узлов по содержимому
   - ✅ Получение статистики графа
   - ✅ Сохранение и загрузка состояния графа
   - ✅ LLM-пайплайн для автоматического извлечения (fallback-режим без API)
   - ✅ REST API endpoints (app/api/routes.py)
   - ⏳ Проактивный агент для анализа (в разработке)

### Примеры использования

#### Добавление узла через Python API

```python
from app.core.repository import GraphRepository
from app.core.models import Node, NodeType

repo = GraphRepository('data/graph')

node = Node(
    content="Машинное обучение — подраздел искусственного интеллекта",
    node_type=NodeType.CONCEPT
)

repo.add_node(node)
repo.save()
```

#### Получение статистики

```python
stats = repo.get_stats()
print(f"Всего узлов: {stats['total_nodes']}")
print(f"Всего связей: {stats['total_edges']}")
```

#### Поиск узлов

```python
results = repo.search_nodes("машинное обучение")
for node in results:
    print(f"- {node.content}")
```

## Хранение данных

Данные хранятся в двух файлах в директории `data/`:

- `graph.gexf` — граф знаний в формате GEXF (поддерживается NetworkX)
- `graph.fragments.json` — исходные фрагменты знаний в JSON

Эти файлы создаются автоматически при первом сохранении.
