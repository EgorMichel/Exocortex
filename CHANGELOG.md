# Changelog

Все заметные изменения в проекте Exocortex будут задокументированы в этом файле.

## [Unreleased]

### Добавлено
- **Базовая структура проекта**: Модульная архитектура с разделением на `core`, `agents`, `llm`, `api`, `services`, `utils`
- **Модели данных** (`app/core/models.py`):
  - `Node`: Узел графа знаний с полями id, тип, содержание, метаданные, параметры памяти (strength, decay_rate, last_interacted)
  - `Edge`: Связь между узлами с полями id, тип, источник, цель, вес, метаданные
  - `NodeType`: Перечисление типов узлов (FACT, CONCEPT, THESIS, DEFINITION, QUESTION, SOURCE)
  - `EdgeType`: Перечисление типов связей (RELATED_TO, CONTRADICTS, SUPPORTS, EXAMPLE_OF, PART_OF, DERIVED_FROM, SIMILAR_TO)
  - `KnowledgeFragment`: Исходный фрагмент знания с метаданными
- **Графовый репозиторий** (`app/core/repository.py`):
  - CRUD операции для узлов и связей
  - Поиск и фильтрация узлов по типу, содержимому
  - Получение «забытых» узлов на основе параметров памяти
  - Навигация по графу (соседние узлы, связанные узлы)
  - Сохранение/загрузка графа в формате GEXF через NetworkX
  - Сохранение/загрузка фрагментов знаний в JSON
  - Статистика графа
- **Интеграция LLM** (`app/llm/extraction.py`):
  - Пайплайн извлечения сущностей и связей из текста
  - Fallback-режим без API (эвристическое извлечение)
  - Поддержка различных типов узлов и связей
- **API слой** (`app/api/routes.py`):
  - REST эндпоинты для управления графом знаний
  - Эндпоинты: GET /nodes, POST /nodes, GET /stats, POST /extract
- **Агент** (`app/agents/`):
  - Заготовка для проактивного агента
- **Конфигурация** (`config/`):
  - Базовые конфигурационные файлы
- **Тесты** (`tests/`):
  - 42 теста для моделей данных, репозитория и LLM-извлечения
  - Покрытие тестами основных операций CRUD
  - Тесты сохранения/загрузки графа

### Изменено
- **Repository initialization**: Добавлена проверка существования файла графа с расширением `.gexf` перед загрузкой
- **Node.from_dict()**: Добавлена обработка служебных атрибутов NetworkX (label)

### Техническое
- **Стек технологий**:
  - Python 3.11+
  - NetworkX для хранения графа в памяти (MVP)
  - Подготовка к интеграции Neo4j для продакшена
  - FastAPI для API слоя
  - pytest для тестирования
